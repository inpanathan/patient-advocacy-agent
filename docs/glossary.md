# Glossary

| Term | Definition |
|---|---|
| **SOAP** | Subjective, Objective, Assessment, Plan — structured clinical note format |
| **ICD-10** | International Classification of Diseases, 10th revision — medical coding system |
| **SCIN** | Skin Condition Image Network — Harvard dermatological dataset (2GB) |
| **Fitzpatrick Type** | Skin classification scale (I-VI) based on response to UV exposure |
| **RAG** | Retrieval-Augmented Generation — combining vector search with LLM generation |
| **WebRTC** | Web Real-Time Communication — browser-based audio/video streaming |
| **STT** | Speech-to-Text — converting spoken audio to text transcription |
| **TTS** | Text-to-Speech — converting text to spoken audio |
| **PII** | Personally Identifiable Information — names, emails, phones, etc. |
| **PHI** | Protected Health Information — medical data linked to a patient |
| **MedGemma** | Google's medical-domain large language model |
| **SigLIP-2** | Google's multimodal (image+text) embedding model |
| **NT-Xent** | Normalized Temperature-scaled Cross-Entropy — contrastive loss function |
| **Escalation** | Flagging a case for immediate physician review (e.g., suspected malignancy) |
| **De-escalation** | Determining that a case is not medical (e.g., paint, tattoo) |
| **ChromaDB** | Open-source vector database for embedding storage and search |
| **Triage** | Initial assessment to determine urgency and appropriate care level |
| **Whisper** | OpenAI's automatic speech recognition model |
| **Pydantic** | Python data validation library using type annotations |
| **FastAPI** | Modern Python web framework for building APIs |
| **structlog** | Structured logging library for Python |
